# Reguli Pentru Titlurile OpenWave

## Principiu
Titlul OpenWave trebuie sa comunice rapid actiunea principala a stirii. Formula de baza este: actor + verb + eveniment.

## Structura standard
- Actorul apare explicit, daca este cunoscut.
- Verbul exprima actiunea reala, nu o formula vaga.
- Evenimentul sau consecinta este formulata concret.

## Exemple de model
- Guvernul aproba noul pachet fiscal
- Primaria Bucuresti extinde zona de transport public
- UE negociaza noi reguli pentru platformele AI

## Reguli de redactare
- Se folosesc verbe active.
- Se evita titlurile abstracte sau criptice.
- Se evita jocurile de cuvinte si formulele de clickbait.
- Se evita caps lock, semnele multiple de exclamare si hiperbola.
- Titlul trebuie sa poata fi citit fluent cu voce tare.

## Cand actorul nu este clar
- Se foloseste institutia, organizatia sau grupul relevant.
- Daca actorul principal este necunoscut, se formuleaza pe baza evenimentului verificat.

## Relatia cu briefingul audio
- Titlul trebuie sa introduca natural rezumatul radio.
- Titlul nu include lista surselor.
- Daca exista controversa, titlul ramane factual si nu forteaza o concluzie.
