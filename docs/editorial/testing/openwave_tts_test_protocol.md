# OpenWave TTS Testing Protocol

## 1. Purpose

The goal of this test is to evaluate how OpenWave briefings sound
when spoken by a synthetic voice.

The test should help identify:

- sentences that sound too long
- unnatural phrasing
- pacing problems
- awkward personalization
- weak story endings

## 2. Pilots to test

Recommended minimum test set:

- Pilot 01 (general news)
- Pilot 02 (service/news mix)
- Pilot 03 (personalized briefing)

## 3. TTS setup

Suggested approach:

- use a high-quality TTS voice
- use the same voice for all pilots
- include intro and outro
- include any news stingers if available

## 4. Listening checklist

During listening, evaluate:

- intro clarity
- natural pronunciation
- sentence length
- quote delivery
- transition flow
- clarity of Two Perspectives
- naturalness of personalization
- strength of the closing line

## 5. Duration check

Record the total audio duration for each pilot.

Compare with the target range:

7-8 minutes.

## 6. Post-listening notes

Document:

- sentences that should be shortened
- phrases that sound artificial
- stories that need stronger endings
- moments where pacing slows down
