# OpenWave Manual Editorial Pilot Workflow

## Purpose
This workflow is used to run a manual editorial simulation before the briefing workflow is implemented in code. The goal is to test editorial judgment, radio flow, story sizing, quote usage, and bulletin structure using real Romanian news.

## Workflow
1. Collect 15 to 20 real news articles from Romanian news sources for the same editorial window.
2. Note obvious duplicate coverage where multiple publications report the same event.
3. Cluster the candidate pool into unique events so the bulletin is built on stories, not articles.
4. Evaluate each cluster for relevance, freshness, and concrete citizen impact.
5. Select the strongest candidate stories and remove items that are repetitive, weak, or too niche for the pilot.
6. Choose the final 8 to 10 stories that create a balanced bulletin.
7. Decide whether one issue genuinely merits a Two Perspectives block based on real editorial contrast.
8. Rewrite each selected story in Romanian radio style, with a clear opening line and natural spoken rhythm.
9. Insert a short quote only when it improves voice, clarity, or immediacy.
10. Assemble the bulletin in running order: intro, standard stories, Two Perspectives block if used, then outro.
11. Review the full bulletin for clarity, pacing, sentence length, and transitions between items.
12. Read the bulletin aloud or test it with TTS to hear rhythm, breath points, and awkward phrasing.
13. Record observations, weak spots, and editorial adjustments for the next pilot run.

## Operating Notes
- Work from clusters, not from single articles.
- Keep the running order varied so heavy topics do not pile up without relief.
- Prefer stories that can be understood quickly on first listen.
- Treat the pilot as an editorial rehearsal, not as a production workflow.
