# Checklist Editorial OpenWave

## Inainte de publicare
- Stirea este confirmata din 2 pana la 4 surse?
- Clusterul trateaza acelasi eveniment si nu amesteca subiecte diferite?
- Titlul respecta formula actor + verb + eveniment?
- Rezumatul are intre 3 si 5 propozitii?

## Claritate audio
- Fiecare propozitie poate fi inteleasa la prima auditie?
- Exista fraze prea lungi sau greu de respirat?
- Sunt evitate acronimele si jargonul neexplicat?
- Textul suna natural cand este citit cu voce tare?

## Calitate jurnalistica
- Impactul asupra oamenilor este explicat devreme?
- Contextul este suficient, dar nu excesiv?
- Exista exagerari, dramatizare sau ambiguitati?
- Eventualele controverse sunt formulate echilibrat?

## Citate si surse
- Citatul, daca exista, este util si scurt?
- Sursele sunt incluse doar in versiunea scrisa?
- Textul audio evita citirea surselor?

## Decizie finala
- Rezumatul este factual, concis si util?
- Povestea merita inclusa in briefingul final?
- Ordinea in briefing reflecta importanta editoriala si relevanta pentru utilizator?
